# User Personas — Productivity Tool for Small Dev Teams

> Last updated: 2026-06-20

## Persona 1: Alex — The Engineering Lead

**Role:** Tech lead / senior engineer at a 8-person startup
**Age:** 28–38
**Technical level:** Strong — ships code daily, reviews PRs, makes architecture calls

### Goals
- Keep the team shipping without becoming the bottleneck
- Know what everyone is working on without asking
- Reduce meeting overhead (standups, syncs, status updates)

### Frustrations
- Jira workflows are too heavy; team spends more time updating status than writing code
- Tools designed for managers, not makers — dashboards for reporting up, not for getting work done
- Context switching between GitHub, Slack/Teams, Notion, and the PM tool

### What would win them over
- "It just works" with zero config — syncs with git automatically
- Keyboard-driven, fast, no clutter
- Async standups built-in (team posts progress, blockers surface automatically)

## Persona 2: Priya — The Full-Stack Engineer

**Role:** Developer at a 12-person agency/product team
**Age:** 24–32
**Technical level:** Strong — owns features end-to-end, frontend + backend

### Goals
- Get into flow state and stay there
- Know what's expected of her without digging through Jira epics
- Spend time on code, not on process

### Frustrations
- Getting pinged in Slack about task updates she already handled
- Over-engineered task fields (story points, sprint points, epic links, etc.)
- Having to manually update statuses that don't reflect reality

### What would win them over
- Task status auto-updates based on real git activity (branch → PR → merge)
- Clean, minimal UI — open a task, see what matters, nothing else
- Smart notifications — only interrupts her when something needs action

## Persona 3: Marcus — The Part-Time PM / Founder

**Role:** CTO or co-founder wearing the PM hat
**Team size:** 5–15
**Background:** Technical founder who now does more planning than coding

### Goals
- Know what's blocking the team without daily standups
- Align the team around a clear focus each week
- Make decisions based on data, not gut feel

### Frustrations
- Current tools surface everything as equally important — no signal/noise separation
- Can't quickly see "what shipped this week" vs "what's stuck"
- Estimating is always wrong, feels like wasted effort

### What would win them over
- Simple "now / next / later" planning instead of rigid sprints
- Automatic cycle analytics: shipped vs planned, cycle time trends, blockers
- Lightweight goals that link to tasks (OKRs without the corporate overhead)

---

## Persona 4: Yuki — The Remote Contractor

**Role:** Part-time frontend contractor working across time zones
**Team size:** Works with 2–3 small teams
**Age:** 25–40
**Technical level:** Senior specialist

### Goals
- Get clear context without sync meetings
- Communicate async without being forgotten
- Onboard quickly to project context

### Frustrations
- Joining a team means learning their tool setup (custom workflows, 47 statuses)
- Being in a different time zone = missing daily standup = being out of the loop
- Hard to know what happened while she was offline

### What would win them over
- Async-first: comments, status updates, and decisions are written, not spoken
- Digest mode: "Here's what happened yesterday" when she opens the tool
- Lightweight context: linked GitHub issues, PRs, and commits tell the story
